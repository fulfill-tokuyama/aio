# AIO Insight — 全体設計オーバービュー

このドキュメントは、AIO Insight の全体像を把握するための設計図である。フロー図・機能一覧・画面一覧を一箇所にまとめ、設計の理解を容易にする。

---

## 1. サービス概要

| 項目 | 内容 |
|------|------|
| サービス名 | AIO Insight |
| 事業者 | フルフィル株式会社（Fulfill Corporation） |
| ターゲット | 日本の中小企業（関西の士業：税理士・社労士・行政書士等） |
| 価値提案 | AI検索（ChatGPT・Perplexity・Gemini等）での可視性を診断し、改善コードを自動生成 |
| 課金モデル | 月額サブスクリプション（¥10,000/月〜） |
| 集客方法 | ① LP経由（検索・広告） ② 診断サマリー付きコールドメール |

---

## 2. 全体フロー図

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          AIO Insight 全体フロー                                    │
└─────────────────────────────────────────────────────────────────────────────────┘

【経路A: LP経由（オーガニック・広告）】

  検索 / SNS / 広告
        │
        ▼
  ┌──────────────┐
  │  /lp         │  ランディングページ
  │  LP          │  ・サービス紹介・料金比較・CTA
  └──────┬───────┘
        │
        ├─ CTA「無料でAI検索可視性を診断する」──▶ /diagnosis
        │
        └─ LP下部 URL入力フォーム ──▶ /diagnosis?url=xxx
                    │
                    ▼
  ┌──────────────┐
  │  /diagnosis │  無料診断ページ
  │  診断実行    │  URL入力 → LLMO診断 → スコア表示（一部ぼかし）
  └──────┬───────┘
        │
        ├─ 「無料登録して詳細を見る」──▶ /signup?diagnosis_id=xxx
        │
        └─ 別URLを診断 ──▶ 診断ページに留まる
                    │
                    ▼
  ┌──────────────┐
  │  /signup     │  新規登録（メールアドレスのみ・Magic Link）
  │  登録        │  メール送信 → リンククリックで認証完了
  └──────┬───────┘
        │
        ▼  auth/callback
  ┌──────────────────────┐
  │  /diagnosis/[id]/    │  詳細診断レポート（認証済み）
  │  detail              │  ・課題全文・改善提案・AI検索別表示・同業種比較
  └──────┬───────────────┘
        │
        └─ 「有料プランに申し込む」──▶ Stripe決済
                    │
                    ▼
  ┌──────────────┐
  │  Stripe      │  決済完了 → Webhook → アカウント作成・ウェルカムメール
  │  有料顧客化   │  → /dashboard（有料機能アンロック）
  └──────────────┘


【経路B: コールドメール経由（営業パイプライン）】

  TAKASHI（運営者）が auto-pipeline 実行
        │
        ▼
  ┌──────────────────┐
  │  URLリスト入力    │  関西士業等のターゲットURL
  └──────┬───────────┘
        │
        ▼
  ┌──────────────────┐
  │  LLMO診断        │  各URLを診断 → diagnosis_reports に保存
  │  フォーム探索     │  contact_email を抽出
  └──────┬───────────┘
        │
        ▼
  ┌──────────────────┐
  │  診断サマリー     │  件名「御社のAI検索対応度を診断しました」
  │  付きメール送信   │  スコア・主な課題 + リンク
  └──────┬───────────┘
        │
        └─ リンク: /signup?diagnosis_id={事前診断済みレポートID}
                    │
                    ▼
  ┌──────────────┐
  │  /signup     │  メールアドレスのみ登録（Magic Link）
  └──────┬───────┘
        │
        ▼
  ┌──────────────────────┐
  │  /diagnosis/[id]/    │  事前診断済みの詳細レポートを即表示
  │  detail              │  （再診断不要）
  └──────┬───────────────┘
        │
        └─ 有料プランCTA ──▶ Stripe決済 ──▶ 有料顧客化

  フォローアップ: Step2(3日後) → Step3(7日後) → Step4(14日後) のメール自動送信
```

## 3. 画面一覧

### 3.1 公開ページ（認証不要）

| パス | 画面名 | 説明 |
|------|--------|------|
| `/` | ハブ | LP・ログイン・ダッシュボードへの導線 |
| `/lp` | ランディングページ | サービス紹介・料金・無料診断CTA・URL入力フォーム |
| `/diagnosis` | 無料診断 | URL入力→診断実行→スコア表示（詳細は登録で解放） |
| `/diagnosis/[id]/detail` | 詳細レポート | 認証済みユーザー向け。課題全文・改善提案・有料CTA |
| `/signup` | 新規登録 | メールアドレスのみ（Magic Link） |
| `/login` | ログイン | メール+パスワード |
| `/privacy` | プライバシーポリシー | 法的必須 |
| `/terms` | 利用規約 | 法的必須 |
| `/tokusho` | 特定商取引法に基づく表記 | 法的必須 |
| `/unsubscribe` | 配信停止 | メール内リンクから遷移 |
| `/payment-success` | 決済完了 | Stripe決済後のサンクスページ |
| `/reset-password` | パスワードリセット | メール送信・新パスワード設定 |

### 3.2 認証必須ページ（顧客・管理者）

| パス | 画面名 | 対象 | 説明 |
|------|--------|------|------|
| `/pipeline` | パイプライン | 管理者 | リード一覧・ステージ管理・メール送信 |
| `/leads` | リード一覧 | 管理者 | 全リードのテーブル表示 |
| `/leads/[id]` | リード詳細 | 管理者 | 個別リード・アクション履歴 |
| `/settings` | 自動化設定 | 管理者 | スキャン・メール・決済設定 |
| `/dashboard` | ダッシュボード | 有料顧客 | スコア推移・Brand Radar・構造化データ生成・meta改善 |

## 4. 機能一覧

### 4.1 ユーザー種別ごとの機能

| 機能 | 未登録 | 無料登録 | 有料顧客 | 管理者 |
|------|:------:|:--------:|:--------:|:------:|
| LP閲覧 | ○ | ○ | ○ | ○ |
| 無料診断（URL入力→スコア） | ○ | ○ | ○ | ○ |
| 詳細レポート閲覧（課題全文・改善提案） | — | ○ | ○ | ○ |
| 構造化データ（JSON-LD）自動生成 | — | — | ○ | ○ |
| metaタグ改善案生成 | — | — | ○ | ○ |
| 月次モニタリングレポート | — | — | ○ | — |
| AI Brand Monitor（6プラットフォーム） | — | — | ○ | — |
| Ahrefs トラフィック・競合分析 | — | — | ○ | — |
| パイプライン管理（リード・メール送信） | — | — | — | ○ |
| 自動化設定・Cron | — | — | — | ○ |

### 4.2 機能カテゴリ別

#### 診断・分析
| 機能 | 説明 | API/実装 |
|------|------|----------|
| LLMO診断 | URL→6カテゴリスコア・弱点・改善提案 | `/api/llmo-scan`, `lib/diagnosis.ts` |
| 構造化データ生成 | JSON-LD（Organization, FAQ等）自動生成 | `/api/improvements/structured-data` |
| meta改善案生成 | title/description のAI改善案3パターン | `/api/improvements/meta-tags` |
| フォーム探索 | URL→contact_email, form_url 抽出 | `/api/scan-forms`, `lib/scan-forms.ts` |
| AI Brand Monitor | ChatGPT/Perplexity/Gemini でのブランド言及チェック | `lib/brand-monitor.ts` |

#### メール
| 機能 | 説明 | テンプレート |
|------|------|-------------|
| 診断結果メール | LP経由の診断申込者へ結果送信 | `lib/email-templates/diagnosis.ts` |
| アウトリーチ（4ステップ） | コールドメール＋フォローアップ | `lib/email-templates/outreach.ts` |
| ウェルカムメール | Stripe決済完了後 | `lib/email-templates/welcome.ts` |
| 月次レポート | 有料顧客へスコア変動・改善提案 | `lib/email-templates/monthly-report.ts` |
| 配信停止 | オプトアウトリンク | `/api/unsubscribe` |

#### 決済・顧客管理
| 機能 | 説明 |
|------|------|
| Stripe決済 | Payment Link → Checkout |
| Webhook | 決済完了→アカウント自動作成→ウェルカムメール |
| 顧客テーブル | `customers`（stripe_customer_id, supabase_user_id） |

#### 営業パイプライン（FormPilot）
| 機能 | 説明 | API |
|------|------|-----|
| 自動パイプライン | URL→診断→保存→フォーム探索→初回メール | `/api/auto-pipeline` |
| 手動メール送信 | リード選択→一括送信 | `/api/auto-send` POST |
| Cron自動送信 | 平日10:00 JST フォローアップ送信 | `/api/auto-send` GET |
| トラッキング | 開封・クリック追跡 | `/api/track` |
| テンプレート統計 | sent / opened / converted | `pipeline_template_stats` |
