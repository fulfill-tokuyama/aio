# パイプライン UI 設計見直し

## 課題

- 何が自動化できていて、何が手動なのか分かりづらい
- 「自動運転中」「次回スキャン」が実際の Cron と一致していない
- 収益最大化のための次のアクションが不明確

## 実施した改善

### 1. 自動化の現状セクション（パイプライン画面トップ）

- **自動で動いている**: フォローアップメール（平日 10:00 JST、Cron）
- **手動実行が必要**: リード発見、LLMO診断、初回メール送信

### 2. 次のアクション（収益最大化）

- 送信待ちがある → 「AIスコア順送信」ボタンで即実行
- フォーム未探索がある → 「自動発見」モーダルを開く
- リード0件 → 「自動発見」で開始を促す

### 3. パイプライン各ステップにラベル追加

| ステップ | ラベル | 説明 |
|---------|--------|------|
| LLMO未対策企業発見 | 手動 | 自動発見 or 一括LLMO調査で実行 |
| フォーム発見 | パイプライン内 | パイプライン実行時に自動検出 |
| 送信待機中 | 手動 | AIスコア順送信で実行 |
| 営業メール送信済 | FU自動 | 初回は手動、フォローアップは平日10時自動 |
| 返信受信 | 手動 | 人間が対応 |
| 有料顧客 | 自動 | Stripe Webhook |

### 4. サイドバー表記の修正

- 「自動運転中」→「フォローアップ自動」（autoSendEnabled に連動）
- 「次回スキャン: 未設定」→「次回FU: 平日 10:00 JST」（Cron の実態に合わせる）

### 5. 自動化設定画面の正確化

- 冒頭に「自動化の仕組み」説明を追加
- LLMO調査: 「手動実行時の設定」に変更（Cron ではないことを明示）
- 自動送信: 「フォローアップ自動送信」に名称変更、Cron であることを明記
- パイプライン全体: 各ステップの自動/手動を正確に表示

## 実際の Cron 一覧（vercel.json）

| パス | スケジュール | 内容 |
|------|-------------|------|
| /api/auto-send | 平日 01:00 UTC (10:00 JST) | フォローアップメール自動送信 |
| /api/rescan | 日曜 18:00 UTC (月曜 03:00 JST) | 有料顧客サイト再診断 |
| /api/cron/brand-check | 毎日 03:00 UTC (12:00 JST) | AIブランドモニタリング |
| /api/cron/monthly-report | 月次 | 有料顧客へ月次レポート |

リード発見・診断・初回送信の Cron は**存在しない**。
