# SME経営者向けコピーライティング — アイデンティティ型スキル

このスキルは、AIO Insight のあらゆる顧客接点テキスト（LP・料金ページ・ヘルプ・エラーメッセージ・診断レポート・メール以外の全テキスト）のトーンと文体を統一する。メール文面については email-templates.md を参照すること。

## インプット
- テキストを書く対象（LP、ヘルプ、UIラベル、エラーメッセージ等）
- 対象ペルソナ（P2: 見込み客 / P3: 契約済み顧客）

## アウトプット
- 本スキルのルールに100%準拠した日本語テキスト

---

## ターゲット像の深掘り

### 典型的な読者
- 50代男性、従業員30名の工務店の社長
- Webサイトは制作会社に作ってもらった。SEOという言葉は知っている
- ChatGPTは名前を知っているが、業務では使っていない
- 「AI検索対策」と言われても、具体的に何をするのか想像できない
- 決済権を持っているが、IT投資は慎重。月額1万円までなら稟議不要
- 毎日忙しく、長い文章は読まない。結論を先に知りたい

### この人に伝わるテキストの条件
- 最初の1文で「何の話か」が分かる
- 自分の業種・状況に当てはまると感じられる
- 専門用語が出てきても（）の補足で意味が分かる
- 数字で裏付けがある（「多くの」ではなく「78%の」）
- 行動が明確（「詳しくはこちら」ではなく「30秒で無料診断する」）

## 文体ルール

### 基本トーン: 「信頼できる専門家が、分かりやすく教えてくれる」

| 要素 | ルール | OK例 | NG例 |
|------|--------|------|------|
| 敬語レベル | 「です・ます」調 | 対策が必要です | 対策が必要でございます |
| 主語 | 読者を主語にする | あなたの会社は… | 弊社のサービスは… |
| 文の長さ | 1文50字以内 | AI検索の利用者は5億人を超えました。 | AI検索の利用者は5億人を超えており、さらに増加傾向にあることから、企業にとって… |
| 段落の長さ | 1段落3文以内 | （3文で改行） | （5文以上の塊） |
| 漢字と平仮名 | 漢字率30-40% | 分かりやすく伝える | 判り易く伝達する |
| 接続詞 | 短いもの優先 | そのため / ただし | したがいまして / しかしながら |

### 数字の表記ルール

| 種類 | ルール | OK例 | NG例 |
|------|--------|------|------|
| 金額 | 円マーク + カンマ区切り | ¥10,000/月 | 月額一万円 |
| 割合 | 半角数字 + % | 78%の企業 | 約8割の企業 |
| 人数 | 半角数字 + 万/億 | 5億人 | 500,000,000人 |
| 時間 | 数字 + 単位 | 30秒で完了 | すぐに完了 |
| 順序 | 丸数字 | ①診断 ②確認 ③開始 | 最初に診断し、次に… |

### 専門用語の扱い

全ての専門用語は、初出時に（）で平易な日本語の補足をつける。2回目以降は補足なしで使用してよい。

| 用語 | 補足表現 |
|------|---------|
| AI検索 | AI検索（ChatGPTやGoogleのAI機能での情報検索） |
| LLMO | LLMO（AIに自社を推薦してもらうための対策） |
| AIO | AIO（AI検索最適化） |
| 構造化データ | 構造化データ（AIがサイト内容を理解するための読み取りガイド） |
| E-E-A-T | E-E-A-T（経験・専門性・権威性・信頼性の4つの評価基準） |
| スコアリング | スコアリング（点数化） |
| パイプライン | パイプライン（見込み客の管理フロー） |
| チャーン | チャーン（解約率） |
| MRR | MRR（月次経常収益） |
| CTR | CTR（クリック率） |

**顧客向けUIでは「LLMO」を使わず「AIO（AI検索最適化）」に統一する。** LLMOは業界用語で顧客には馴染みがない。内部ダッシュボード（FormPilot）ではLLMOを使ってよい。

## テキスト種別ごとの書き方

### 1. LP（ランディングページ）

**目的**: 見込み客に「AI検索対策が必要だ」と気づかせ、無料診断に誘導する

**構成原則**:
- 結論（ヘッドライン）→ 理由（データ）→ 自分ごと化 → 行動（CTA）
- 1セクション = 1メッセージ。複数のメッセージを詰め込まない
- CTAは最低3箇所（ファーストビュー・中間・最下部）

**ヘッドラインの型**:
```
型A【問いかけ型】: 「AIに聞いて、あなたの会社は出てきますか？」
型B【数字衝撃型】: 「AI検索の利用者5億人。あなたの見込み客はもう使っています」
型C【危機感型】  : 「AI検索で見つからない会社は、存在しないのと同じです」
型D【解決提示型】: 「月額1万円で、AI検索からの集客を始めませんか」
```

**使ってよいLP表現**:
- 「無料で診断」「30秒で完了」（ハードルを下げる）
- 「○○業界の経営者様に選ばれています」（具体的数字がある場合のみ）
- 「まずは現状を知ることから」（行動の第一歩を軽くする）

**使ってはいけないLP表現**:
- 「業界No.1」「日本初」（根拠なしの最上級表現）
- 「今だけ」「残り○名」（虚偽の希少性。信頼を毀損する）
- 「簡単」「誰でもできる」（実際の手間と乖離する表現）
- 「AIで全自動」（AIの能力を過度に宣伝する表現）

### 2. 料金ページ

**目的**: 費用対効果を納得してもらい、プラン選択→決済に進めてもらう

**書き方ルール**:
- 最安プランを最初に見せる（月額1万円の心理的閾値を先にクリアさせる）
- 各プランの違いは「できること」で表現する（「○○機能」ではなく「AIスコアが毎月届く」）
- 「おすすめ」バッジは1プランだけにつける（選択肢を絞る）
- 年払いの割引がある場合は「年払いで2ヶ月分お得」のように具体的に書く

**料金表のラベル例**:
```
NG: 「AI可視性レポート機能」
OK: 「毎月届くAI可視性レポート」

NG: 「構造化データ診断」
OK: 「AIがあなたのサイトを理解しやすくなっているかの診断」

NG: 「E-E-A-T改善提案」
OK: 「信頼性を高めるための具体的な改善提案」
```

### 3. ヘルプ・ガイド

**目的**: 操作方法や用語の意味を、迷わず理解できるようにする

**書き方ルール**:
- タイトルは「○○するには」の形式にする（「スキャン設定について」ではなく「スキャン対象を変更するには」）
- 手順は番号付きリスト。各手順は1アクション=1文
- スクリーンショットを添える場合は、操作箇所を赤枠で囲む
- 「もし〜の場合」の分岐は最大2つまで。3つ以上は別記事に分割する

**ヘルプ記事テンプレート**:
```
# [操作名]するには

## この機能について
[1-2文で機能の目的を説明]

## 手順
① [画面名]を開きます
② [ボタン名/メニュー名]をクリックします
③ [入力内容]を入力します
④ [確定ボタン]をクリックして完了です

## うまくいかない場合
- [よくある問題1]: [対処法]
- [よくある問題2]: [対処法]
```

### 4. UIテキスト（ラベル・ボタン・プレースホルダー）

**ボタンテキスト**:
| 場面 | テキスト | NGテキスト |
|------|---------|-----------|
| 主要アクション（CTA） | 無料で診断する | Submit / 送信 |
| 保存 | 保存する | Save |
| キャンセル | キャンセル | やめる / 戻る |
| 削除 | 削除する | Delete |
| 次へ | 次のステップへ | Next |
| 開始 | スキャンを開始 | 開始 / Start |

**ルール**:
- ボタンは「○○する」の動詞形にする（名詞だけにしない）
- 英語のラベルは使わない（Submit → 送信する、Cancel → キャンセル）
- 破壊的アクション（削除等）は確認ダイアログを挟む。ダイアログ文面は「本当に[対象]を削除しますか？この操作は取り消せません。」

**プレースホルダー**:
```
OK: 「例: 株式会社〇〇」「例: https://example.com」
NG: 「企業名を入力」「URLを入力してください」
```
プレースホルダーは具体例を見せる。「〇〇を入力」は情報量ゼロ。

### 5. エラーメッセージ

**3つのルール**:
1. 何が起きたかを伝える（事実）
2. なぜ起きたかを伝える（原因）
3. どうすればいいかを伝える（対処法）

**テンプレート**:
```
[何が起きたか]。[原因]。[対処法]。

例: メールの送信に失敗しました。送信先のメールアドレスが無効な可能性があります。
    リード詳細画面でメールアドレスを確認してください。

例: スキャンを実行できませんでした。1日のスキャン上限（50社）に達しています。
    明日以降に再度お試しください。
```

**NG例**:
```
× 「エラーが発生しました」（何が起きたか不明）
× 「Error: NETWORK_TIMEOUT」（英語のまま）
× 「入力値が不正です」（何が不正か不明）
× 「しばらくお待ちください」（いつまで待てばいいか不明）
```

### 6. 診断レポート

**目的**: 診断スコアと改善ポイントを、経営者が社内で共有できるレベルで分かりやすく伝える

**書き方ルール**:
- 最初に結論（スコアとランク）を大きく見せる
- 各項目のスコアは「○点/○点」+ 棒グラフで視覚化する
- 改善ポイントは「現状」→「対策」→「効果」の3要素で書く
- 業種別の平均スコアとの比較を入れる（自分ごと化）
- レポート末尾に「改善のご相談はこちら」CTAを配置する

**改善ポイントの書き方テンプレート**:
```
■ [項目名]（[現在の点数]/[満点]）

現状: [具体的な問題点を1文で]
対策: [具体的にやるべきことを1文で]
効果: [対策するとどう変わるかを1文で]

例:
■ 構造化データ（5点/20点）

現状: サイトにJSON-LD（AIがサイト内容を理解するためのコード）が設置されていません。
対策: 会社情報・サービス内容・よくある質問のJSON-LDを追加します。
効果: AIが御社のサービス内容を正しく理解し、推薦に含まれやすくなります。
```

- [ ] 最初の1文で「何の話か」が分かるか
- [ ] 1文50字以内に収まっているか
- [ ] 専門用語の初出に（）の補足があるか
- [ ] 数字は具体的か（「多くの」ではなく「78%」）
- [ ] CTAは「○○する」の動詞形になっているか
- [ ] 英語がそのまま残っていないか
- [ ] 読者の主語で書かれているか（「弊社は」ではなく「あなたは」）
- [ ] 1段落3文以内に収まっているか
- [ ] 顧客向けUIで「LLMO」ではなく「AIO」を使っているか
