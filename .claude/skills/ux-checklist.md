# UX評価チェックリスト — 辞書型スキル

このスキルは、AIO Insight（FormPilot）のUIをレビューする際の評価基準を定義する。レビュアーエージェントはこのチェックリストに基づいて0-5点で採点し、合計90点以上（100点満点）を合格とする。

## インプット
- レビュー対象の画面名とスクリーンショット（Playwright CLI で撮影）
- 実装された機能の概要

## アウトプット
- 各項目の0-5点スコアと具体的な指摘コメント
- 合計点と合否判定
- 不合格の場合の改善優先順位（最も影響が大きい項目から順に）

## 重要: コンテキストフレッシュ運用
このチェックリストを使ってレビューする際は、**必ず新しいレビュアーエージェントで評価する**こと。同一レビュアーで繰り返すとスコアが甘くなるバイアスが発生する。

## 評価項目（全20項目 × 5点 = 100点満点）

### カテゴリ1: 第一印象・視覚品質（4項目 / 20点）

#### 1-1. 視覚的一貫性（5点）
design-tokens.md に定義されたカラー・余白・角丸・フォントサイズに準拠しているか。

| 点数 | 基準 |
|------|------|
| 5 | 全要素がトークンに100%準拠 |
| 4 | 1-2箇所の軽微な逸脱 |
| 3 | 3-5箇所の逸脱、または色の使い方に一部不統一 |
| 2 | 全体的にトークンが守られていない |
| 1 | デザイントークンを無視している |
| 0 | 未実装 or 表示崩れ |

#### 1-2. 情報の視認性（5点）
テキストが読みやすく、重要な情報がすぐ目に入るか。

| 点数 | 基準 |
|------|------|
| 5 | 視線の流れが自然で、重要情報が3秒以内に認識できる |
| 4 | 概ね読みやすいが、一部の情報が埋もれている |
| 3 | テキストサイズやコントラストに改善の余地がある |
| 2 | 重要な情報を見つけるのに時間がかかる |
| 1 | テキストが読みにくい、またはコントラスト不足 |
| 0 | 内容が判読できない |

#### 1-3. レイアウトの整然さ（5点）
グリッドの整合性、余白の均一性、要素の配置バランス。

| 点数 | 基準 |
|------|------|
| 5 | グリッドが完璧に揃い、余白が均一 |
| 4 | 概ね整然だが、1-2箇所で余白の不揃いがある |
| 3 | 配置にバラつきがあり、整理の余地がある |
| 2 | レイアウトが乱れている部分が目立つ |
| 1 | 要素の配置に一貫性がない |
| 0 | レイアウト崩れで使用不能 |

#### 1-4. ブランド適合性（5点）
FormPilotらしいダークテーマ・プロフェッショナルな雰囲気が出ているか。

| 点数 | 基準 |
|------|------|
| 5 | 既存画面と並べても完全に統一感がある |
| 4 | ほぼ統一されているが、微妙なトーンの違いがある |
| 3 | FormPilotらしさが薄い部分がある |
| 2 | 既存画面との差異が目立つ |
| 1 | 別のサービスに見える |
| 0 | デザインが未適用 |

---

### カテゴリ2: 操作性・インタラクション（5項目 / 25点）

#### 2-1. 主要アクションの明確さ（5点）
この画面で最も重要な操作が、ひと目で分かるか。

| 点数 | 基準 |
|------|------|
| 5 | CTAが1つ明確に強調され、次にやるべきことが迷わず分かる |
| 4 | CTAは分かるが、競合する要素がやや気になる |
| 3 | 複数のアクションが同じ強度で表示され、優先順位が曖昧 |
| 2 | 何をすればいいか分かるまでに10秒以上かかる |
| 1 | CTAが見つけにくい |
| 0 | アクションが不明 |

#### 2-2. フィードバックの即時性（5点）
操作した結果がすぐにユーザーに伝わるか。

| 点数 | 基準 |
|------|------|
| 5 | 全操作に即座のフィードバック（ローディング・成功・エラー）がある |
| 4 | 主要操作にはフィードバックがあるが、一部欠けている |
| 3 | ローディング状態はあるが、成功/エラーの表示が不十分 |
| 2 | フィードバックが遅い、または分かりにくい |
| 1 | 操作後に何が起きたか分からない場面がある |
| 0 | フィードバックが一切ない |

#### 2-3. エラーハンドリング（5点）
エラー時にユーザーが困らないか。

| 点数 | 基準 |
|------|------|
| 5 | 全エラーに日本語の具体的メッセージ + 対処法が表示される |
| 4 | エラーメッセージはあるが、対処法が一部不明確 |
| 3 | エラーメッセージが技術的すぎる、または英語のまま |
| 2 | 一部のエラーでメッセージが表示されない |
| 1 | エラー時に画面が固まる、または白画面になる |
| 0 | エラーハンドリングが未実装 |

#### 2-4. ナビゲーションの直感性（5点）
現在地が分かるか、目的の画面にすぐたどり着けるか。

| 点数 | 基準 |
|------|------|
| 5 | 現在地が明確で、2クリック以内にどの画面にも到達可能 |
| 4 | 概ね良好だが、一部の画面への導線が分かりにくい |
| 3 | 現在地の表示が不明確、またはナビが一部不足 |
| 2 | 目的の画面にたどり着くのに迷う |
| 1 | ナビゲーションが機能していない |
| 0 | ナビゲーション未実装 |

#### 2-5. フォームの使いやすさ（5点）
入力フォームがストレスなく使えるか（該当なしの場合は自動5点）。

| 点数 | 基準 |
|------|------|
| 5 | バリデーション即時表示・Tab移動対応・プレースホルダー適切 |
| 4 | 主要なバリデーションはあるが、UXの改善余地あり |
| 3 | バリデーションが送信後にのみ表示される |
| 2 | フォームの入力でストレスを感じる部分がある |
| 1 | バリデーションが不十分で、エラー多発 |
| 0 | フォームが動作しない |

---

### カテゴリ3: 情報設計・コンテンツ（4項目 / 20点）

#### 3-1. 情報の過不足（5点）
この画面に必要な情報が揃っているか、不要な情報が混じっていないか。

| 点数 | 基準 |
|------|------|
| 5 | 必要な情報が過不足なく、優先順位通りに配置されている |
| 4 | 概ね良いが、1-2項目の追加または削除が望ましい |
| 3 | 情報の過不足、または優先順位のズレがある |
| 2 | 重要な情報が欠けている、または不要な情報が多い |
| 1 | 画面の目的に対して情報設計が不適切 |
| 0 | 必要な情報が表示されていない |

#### 3-2. ステータスの明確さ（5点）
データの状態（未処理・処理中・完了・エラー等）が色+アイコン+テキストで伝わるか。

| 点数 | 基準 |
|------|------|
| 5 | 全ステータスが色+アイコン+テキストの3要素で表現されている |
| 4 | ほぼ3要素だが、一部のステータスで要素が欠けている |
| 3 | 色だけ、またはテキストだけでステータスを伝えている |
| 2 | ステータスの区別が分かりにくい |
| 1 | ステータスが表示されていない箇所がある |
| 0 | ステータス表示が未実装 |

#### 3-3. 専門用語の扱い（5点）
LLMO/AIO等の専門用語に、平易な日本語の補足があるか。

| 点数 | 基準 |
|------|------|
| 5 | 全ての専門用語に（）やツールチップで補足説明がある |
| 4 | 主要な用語には補足があるが、一部漏れている |
| 3 | 補足はあるが、説明が分かりにくい |
| 2 | 専門用語がそのまま使われている箇所が多い |
| 1 | 補足説明がほぼない |
| 0 | 英語の技術用語がそのまま表示されている |

#### 3-4. 空状態（Empty State）の設計（5点）
データがゼロの場合に適切なガイダンスが表示されるか。

| 点数 | 基準 |
|------|------|
| 5 | 空状態に説明+イラスト/アイコン+次のアクションCTAがある |
| 4 | 説明とCTAはあるが、視覚的な工夫が不足 |
| 3 | 「データがありません」のみ表示 |
| 2 | 空状態の表示がない（白い空間が残る） |
| 1 | エラーメッセージが表示される |
| 0 | 空状態で画面が壊れる |

### カテゴリ4: パフォーマンス・信頼性（4項目 / 20点）

#### 4-1. 表示速度体感（5点）
| 点数 | 基準 |
|------|------|
| 5 | 画面遷移・データ表示が1秒以内で完了する |
| 4 | 2秒以内で概ね問題ないが、一部の重い処理あり |
| 3 | 3秒以上かかる処理があるが、ローディング表示あり |
| 2 | 5秒以上かかる処理があり、体感的に遅い |
| 1 | 表示に10秒以上かかる |
| 0 | タイムアウトまたはフリーズする |

#### 4-2. レスポンシブ対応（5点）
| 点数 | 基準 |
|------|------|
| 5 | デスクトップ・タブレット・モバイルで完璧に動作する |
| 4 | 全デバイスで使用可能だが、モバイルで一部レイアウト崩れ |
| 3 | デスクトップは良好だが、モバイルで操作しにくい部分がある |
| 2 | モバイルで重大なレイアウト崩れがある |
| 1 | モバイルでは使用困難 |
| 0 | デスクトップでも表示崩れがある |

#### 4-3. データの整合性（5点）
| 点数 | 基準 |
|------|------|
| 5 | 表示データがSupabaseと完全に一致、リアルタイム反映 |
| 4 | データは正確だが、更新にリロードが必要な箇所がある |
| 3 | 一部のデータで表示と実データにタイムラグがある |
| 2 | データの不整合が確認される |
| 1 | 表示データが信頼できない |
| 0 | データが表示されない |

#### 4-4. セキュリティ基本事項（5点）
| 点数 | 基準 |
|------|------|
| 5 | 認証チェック・入力サニタイズ・CORS設定が適切 |
| 4 | 主要なセキュリティ対策はあるが、一部のエッジケースが未処理 |
| 3 | 基本的な認証はあるが、入力バリデーションが不十分 |
| 2 | セキュリティ上の懸念がある |
| 1 | 未認証でダッシュボードにアクセスできる |
| 0 | 重大なセキュリティホールがある |

---

### カテゴリ5: 安心感・わかりやすさ（3項目 / 15点）

#### 5-1. 安心感の演出（5点）
中小企業経営者が「このツールなら任せられる」と感じるか。

| 点数 | 基準 |
|------|------|
| 5 | 実績数値・ステータス表示・操作ガイドが充実し、プロフェッショナルな印象 |
| 4 | 概ね安心感があるが、一部の画面でガイダンスが不足 |
| 3 | 機能は動くが、「信頼できるツール」という印象が弱い |
| 2 | 素人っぽさ、または不安を感じる要素がある |
| 1 | ベータ版のような印象を受ける |
| 0 | ツールとしての信頼性を感じない |

#### 5-2. 次のステップの明確さ（5点）
ユーザーが「次に何をすればいいか」常に分かるか。

| 点数 | 基準 |
|------|------|
| 5 | どの画面でも次のアクションが明確で、迷う場面がゼロ |
| 4 | 主要フローでは明確だが、サブフローで一部不明確 |
| 3 | 一部の画面で「次に何をすればいいか」が分かりにくい |
| 2 | 複数の画面で操作に迷う |
| 1 | ほとんどの画面で次のステップが不明確 |
| 0 | ユーザーが放置される |

#### 5-3. 日本語品質（5点）
| 点数 | 基準 |
|------|------|
| 5 | 全テキストが自然な日本語で、敬語レベルも統一されている |
| 4 | 概ね良好だが、1-2箇所で不自然な表現がある |
| 3 | 機械翻訳的な表現、または敬語の不統一がある |
| 2 | 英語がそのまま残っている箇所が複数ある |
| 1 | 日本語として違和感がある箇所が多い |
| 0 | 英語UIのまま |
