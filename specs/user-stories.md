# AIO Insight（FormPilot）ユーザーストーリー集

このファイルは、AIO Insight の全機能を「誰が・何のために・何をするか」で定義する。新機能の実装や既存機能の改修を行う際、必ずこのファイルの該当ストーリーを参照し、ストーリーに記載のないユースケースは先にこのファイルに追記してから着手すること。

---

## ペルソナ定義

### P1: TAKASHI（サービス運営者）
- フルフィル株式会社 CSO
- AIO Insight の企画・開発・運営を統括
- FormPilot のダッシュボードを毎日確認する
- 目的: パイプラインの自動化率を最大化し、ゼロミーティングで受注まで回す

### P2: 見込み客（SME経営者）
- 日本の中小企業経営者（従業員10-100名）
- 年齢40-60代、ITリテラシー中程度
- AI検索の存在は知っているが、自社への影響を実感していない
- 目的: 自社の集客を改善したいが、何をすればいいか分からない

### P3: 契約済み顧客
- 無料診断を経て有料プラン（月額1万円〜）に契約した経営者
- AI検索対策の必要性は理解済み
- 目的: 投資に見合う効果を実感したい。毎月のレポートで成果を確認したい

### P4: 将来ペルソナ — 代理店パートナー（Phase3以降）
- Web制作会社やSEOコンサルタント
- 自社クライアントにAIO対策を提案したい
- 目的: AIO Insight を自社サービスに組み込んで顧客に提供する

## LP・集客フロー（P2: 見込み客）

### US-LP-01: サービスを初めて知る
```
ペルソナ: P2（見込み客）
画面: LP-01（/lp）
ストーリー:
  Google検索やSNS広告からLPにたどり着いた経営者として、
  AI検索対策が自社に必要かどうかを3分以内に理解したい。
  なぜなら、忙しいので長い説明を読む時間がないから。
受け入れ条件:
  - ファーストビューで「何のサービスか」が5秒で分かる
  - 危機感セクションでAI検索の重要性をデータ付きで理解できる
  - 自分ごと化セクションで「うちも対策が必要かも」と感じる
  - スクロールなしでCTA（無料診断）が視界に入る
```

### US-LP-02: 無料診断を受ける
```
ペルソナ: P2（見込み客）
画面: LP-02（/lp#diagnosis）→ LP-03（/lp/result）
ストーリー:
  自社のAI検索での可視性を知りたい経営者として、
  30秒以内に診断を完了し、結果を即座に確認したい。
  なぜなら、手間がかかるなら途中で離脱するから。
受け入れ条件:
  - 入力項目は最小限（企業名 + URL + 業種 + メールアドレス）
  - 入力から結果表示まで30秒以内
  - 結果がスコア（0-100）+ ランク（A-E）で一目で分かる
  - 結果ページに「もっと詳しく知る」CTAがある
  - メールアドレスを取得し、Supabase の leads テーブルにリード登録する
```

### US-LP-03: 無料診断結果から有料プランを検討する
```
ペルソナ: P2（見込み客）
画面: LP-03（/lp/result）
ストーリー:
  診断スコアがC以下だった経営者として、
  具体的に何をすればスコアが上がるかを知り、
  有料プランの費用対効果を判断したい。
  なぜなら、課題は分かったが解決策と費用感が分からないと動けないから。
受け入れ条件:
  - 改善ポイントが具体的に3つ以上表示される
  - 各改善ポイントに「対策すると何が変わるか」の説明がある
  - 料金プラン（月額1万円〜）が明記されている
  - 「月額1万円で始める」CTAがある
  - Stripe決済に直接遷移できる（ゼロミーティング）
```

## 自動営業パイプライン（P1: 運営者）

### US-PIPE-01: パイプライン全体を俯瞰する
```
ペルソナ: P1（TAKASHI）
画面: DASH-01（/pipeline）
ストーリー:
  毎朝ダッシュボードを開く運営者として、
  パイプラインの全体状況を10秒で把握したい。
  なぜなら、どのステージにボトルネックがあるか即座に判断し、
  自動化の設定を調整する必要があるから。
受け入れ条件:
  - 各ステージ（発見→キュー→送信済→商談→受注）のリード数が一覧で見える
  - MRR（月次経常収益）が画面上部に常時表示される
  - 自動化ステータス（ON/OFF）が分かる
  - 直近24時間のアクティビティサマリーが表示される
  - リードカードをクリックすると詳細に遷移できる（→ DASH-03）
```

### US-PIPE-02: 新規スキャンを実行する
```
ペルソナ: P1（TAKASHI）
画面: DASH-01（/pipeline）→ SET-01（/settings/scan）
ストーリー:
  新しい業種・地域のリードを開拓したい運営者として、
  スキャン対象を設定してワンクリックでスキャンを実行したい。
  なぜなら、手動でリードを探す時間を省きたいから。
受け入れ条件:
  - 対象業種をチェックボックスで選択できる
  - 対象地域を選択できる
  - 「スキャン開始」ボタンでスキャンが即座に実行される
  - スキャン中の進捗（○○社中○○社完了）がリアルタイムで見える
  - 完了時にトースト通知で「○社の新規リードを発見」と表示される
```

### US-PIPE-03: リードの詳細を確認しアクションする
```
ペルソナ: P1（TAKASHI）
画面: DASH-03（/leads/[id]）
ストーリー:
  特定のリードに対して手動でアクションを取りたい運営者として、
  そのリードの診断スコア・アクション履歴・連絡先情報を
  1画面で確認し、次のアクションを選びたい。
  なぜなら、高スコアのリードには個別対応が効果的だから。
受け入れ条件:
  - 企業名・URL・業種・地域・スコア・ランクが上部に表示される
  - スキャン詳細（各項目の点数と改善ポイント）が確認できる
  - メール送信履歴（日時・件名・開封/クリック状況）が時系列で見える
  - 「メール送信」「ステージ変更」「メモ追加」のアクションが実行できる
  - 前のリスト画面に戻るボタンがある
```

### US-PIPE-04: メールのA/Bテストを設定・確認する
```
ペルソナ: P1（TAKASHI）
画面: DASH-05（/ab-test）
ストーリー:
  メールの開封率を改善したい運営者として、
  件名やCTA文言のA/Bテストを設定し、結果を確認したい。
  なぜなら、データに基づいてメール品質を継続改善したいから。
受け入れ条件:
  - テスト名・バリアント（A/B）の件名と本文を入力できる
  - テスト対象リード数の配分を設定できる
  - 実行中テストの進捗（開封率・クリック率の差）がリアルタイムで見える
  - 統計的有意差に達したら自動で勝者判定される
  - 勝者バリアントをデフォルトテンプレートに反映するボタンがある
```

### US-PIPE-05: フォローアップ対象を管理する
```
ペルソナ: P1（TAKASHI）
画面: DASH-06（/followup）
ストーリー:
  フォローアップが必要なリードを効率的に管理したい運営者として、
  未開封・開封済み未クリック・興味ありのリードを
  フォローアップ状況別に一覧で確認したい。
  なぜなら、適切なタイミングで適切なフォローを送りたいから。
受け入れ条件:
  - フォローアップ対象リードがステータス別にフィルタできる
  - 各リードの「最終接触日」「フォローアップ回数」が見える
  - 一括フォローアップ送信ボタンがある
  - 個別にフォローアップを停止/再開できる
```

## 分析・収益管理（P1: 運営者）

### US-ANA-01: パフォーマンスを分析する
```
ペルソナ: P1（TAKASHI）
画面: DASH-07（/analytics）
ストーリー:
  パイプラインのパフォーマンスを数値で把握したい運営者として、
  スキャン数→リード数→送信数→開封率→商談率→受注率の
  ファネル全体を週次・月次で分析したい。
  なぜなら、どのステージの改善が最もROIが高いかを判断するため。
受け入れ条件:
  - ファネルチャート（各ステージの件数と転換率）が表示される
  - 期間フィルタ（7日/30日/90日/カスタム）で切り替え可能
  - メールの開封率・クリック率・返信率のトレンドグラフがある
  - 業種別・地域別のパフォーマンス比較ができる
  - A/Bテスト結果のサマリーが表示される
```

### US-ANA-02: 収益とMRRを管理する
```
ペルソナ: P1（TAKASHI）
画面: DASH-08（/customers）
ストーリー:
  サービスの収益状況を把握したい運営者として、
  MRR・契約顧客数・プラン別内訳・チャーン率を確認したい。
  なぜなら、事業の健全性を常にモニタリングする必要があるから。
受け入れ条件:
  - MRR（月次経常収益）が大きく表示される
  - 契約顧客の一覧（企業名・プラン・契約日・最終更新日）が表示される
  - プラン別の契約数内訳が見える
  - Stripe の決済状況（成功・失敗・解約）がリアルタイム反映される
  - 今月の新規契約数・解約数が分かる
```

## 契約済み顧客の体験（P3: 契約済み顧客）

### US-CUST-01: 月次レポートを受け取る
```
ペルソナ: P3（契約済み顧客）
画面: メール受信 → 顧客ダッシュボード（Phase2で実装）
ストーリー:
  毎月の対策効果を知りたい契約顧客として、
  AI可視性スコアの推移と改善状況を分かりやすく確認したい。
  なぜなら、月額料金を払い続ける価値があるかを判断したいから。
受け入れ条件:
  - 毎月1日にPDF/HTMLレポートがメールで届く
  - スコアの前月比（上がったか下がったか）が一目で分かる
  - 実施した対策の一覧と、その効果が紐付いて表示される
  - 次月の推奨アクションが提示される
  - レポートの中に専門用語があれば（）で補足されている
```

### US-CUST-02: 契約プランを変更する
```
ペルソナ: P3（契約済み顧客）
画面: 顧客設定画面（Phase2で実装）
ストーリー:
  効果を実感してプランをアップグレードしたい顧客として、
  管理画面からプラン変更を自分で完了したい。
  なぜなら、電話やメールで連絡する手間を省きたいから。
受け入れ条件:
  - 現在のプランと他プランの比較が表示される
  - 「プラン変更」ボタンでStripe決済が即座に更新される
  - 日割り計算の説明がある
  - 変更完了の確認メールが届く
```

## 自動化設定（P1: 運営者）

### US-SET-01: スキャン対象を設定する
```
ペルソナ: P1（TAKASHI）
画面: SET-01（/settings/scan）
ストーリー:
  スキャン対象の業種・地域を柔軟に変更したい運営者として、
  チェックボックスで対象を選び、スキャン頻度を設定したい。
  なぜなら、高優先度の業種に集中してリードを獲得したいから。
受け入れ条件:
  - 業種リスト（ブライダル、美容医療、不動産、士業、飲食 等）が選択できる
  - 地域（都道府県 or 主要都市）が選択できる
  - スキャン頻度（毎日/週1/月1/手動のみ）が設定できる
  - 1回あたりの最大スキャン数が設定できる
  - 設定変更が即座に保存される
```

### US-SET-02: メール送信設定を管理する
```
ペルソナ: P1（TAKASHI）
画面: SET-02（/settings/email）
ストーリー:
  メールの送信元情報やテンプレートを管理したい運営者として、
  送信者名・返信先・テンプレート文面を設定画面から変更したい。
  なぜなら、コードを触らずにメール内容を改善したいから。
受け入れ条件:
  - 送信者名・送信元メールアドレス・返信先が設定できる
  - メールテンプレート（初回/フォローアップ/診断結果/商談誘導）を編集できる
  - テンプレートのプレビューが見られる
  - 変数（[企業名][業種]等）のプレースホルダーが使える
  - 1日の送信上限が設定できる
```

## ストーリー追加ルール

1. 新機能を実装する前に、必ず該当するユーザーストーリーをこのファイルに追記する
2. ストーリーのID命名規則: US-[カテゴリ]-[連番]（例: US-PIPE-06）
3. ペルソナは必ずP1-P4のいずれかを指定する
4. 受け入れ条件は具体的・検証可能な表現で書く（「見やすい」ではなく「5秒以内に認識できる」）
5. 画面IDは specs/flow.md の画面一覧と一致させる
6. Phase2以降の機能は、ストーリー内にその旨を明記する
