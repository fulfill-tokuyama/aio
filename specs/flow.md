# AIO Insight（FormPilot）画面遷移図 — Flow.md

このファイルは、AIO Insight の全画面とその遷移関係を定義する。新しい画面を追加する場合、必ずこのファイルに遷移元・遷移先を追記すること。「画面は作ったが、どこからたどり着くかの導線がない」問題を防ぐための設計図である。

---

## 画面一覧

| ID | パス | 画面名 | 認証 | 説明 |
|----|------|--------|------|------|
| LP-01 | /lp | ランディングページ | 不要 | サービス紹介・無料診断CTA |
| LP-02 | /lp#diagnosis | 無料診断フォーム | 不要 | LP下部の診断フォーム（アンカーリンク） |
| LP-03 | /lp/result | 診断結果ページ | 不要 | 無料診断の結果表示・有料プランCTA |
| AUTH-01 | /login | ログイン | 不要 | メール/パスワード or Google認証 |
| AUTH-02 | /signup | 新規登録 | 不要 | アカウント作成（診断結果からの導線あり） |
| DASH-01 | /pipeline | パイプライン | 必要 | メイン画面。リードのステージ別管理 |
| DASH-02 | /leads | リード一覧 | 必要 | 全リードのテーブル表示・検索・フィルタ |
| DASH-03 | /leads/[id] | リード詳細 | 必要 | 個別リードの詳細情報・アクション履歴 |
| DASH-04 | /settings | 自動化設定 | 必要 | スキャン頻度・メールテンプレート・API設定 |
| DASH-05 | /ab-test | A/Bテスト | 必要 | メールのA/Bテスト管理・結果表示 |
| DASH-06 | /followup | フォローアップ | 必要 | フォローアップ対象リード・再送管理 |
| DASH-07 | /analytics | 分析 | 必要 | パフォーマンスメトリクス・チャート |
| DASH-08 | /customers | 顧客・収益 | 必要 | Stripe連携・MRR・契約顧客管理 |
| DASH-09 | /activity | アクティビティ | 必要 | 全アクションのタイムライン表示 |
| SET-01 | /settings/scan | スキャン設定 | 必要 | スキャン対象業種・地域・頻度の設定 |
| SET-02 | /settings/email | メール設定 | 必要 | 送信元・テンプレート・SendGrid/Resend設定 |
| SET-03 | /settings/stripe | 決済設定 | 必要 | Stripeキー・プラン・Webhook設定 |

## 遷移図（メインフロー）

```
[外部]
  │
  ├─ Google検索 / SNS / 広告
  │     │
  │     ▼
  │   LP-01 ランディングページ (/lp)
  │     │
  │     ├─ CTAクリック ──▶ LP-02 無料診断フォーム (/lp#diagnosis)
  │     │                    │
  │     │                    ▼
  │     │                 LP-03 診断結果 (/lp/result)
  │     │                    │
  │     │                    ├─ 「詳しく見る」──▶ AUTH-02 新規登録 (/signup)
  │     │                    └─ 「無料で始める」──▶ AUTH-02 新規登録 (/signup)
  │     │
  │     └─ ヘッダー「ログイン」──▶ AUTH-01 ログイン (/login)
  │
  └─ 直接アクセス（既存ユーザー）
        │
        ▼
      AUTH-01 ログイン (/login)
        │
        ├─ 認証成功 ──▶ DASH-01 パイプライン (/pipeline)
        └─ 未登録 ──▶ AUTH-02 新規登録 (/signup)
                          │
                          └─ 登録完了 ──▶ DASH-01 パイプライン (/pipeline)
```

## 遷移図（ダッシュボード内）

```
DASH-01 パイプライン (/pipeline)  ◀── デフォルトのホーム画面
  │
  ├─ サイドナビ（常時表示）
  │     ├─ 「パイプライン」──▶ DASH-01
  │     ├─ 「リード一覧」──▶ DASH-02
  │     ├─ 「自動化設定」──▶ DASH-04
  │     ├─ 「A/Bテスト」──▶ DASH-05
  │     ├─ 「フォローアップ」──▶ DASH-06
  │     ├─ 「分析」──▶ DASH-07
  │     ├─ 「顧客・収益」──▶ DASH-08
  │     └─ 「アクティビティ」──▶ DASH-09
  │
  ├─ パイプラインカード クリック ──▶ DASH-03 リード詳細 (/leads/[id])
  │
  └─ 「スキャン実行」ボタン ──▶ スキャン開始（DASH-01上でインライン表示）

DASH-02 リード一覧 (/leads)
  │
  ├─ リード行クリック ──▶ DASH-03 リード詳細 (/leads/[id])
  └─ 「新規スキャン」──▶ SET-01 スキャン設定 (/settings/scan)

DASH-03 リード詳細 (/leads/[id])
  │
  ├─ 「メール送信」──▶ メール作成モーダル（DASH-03上）
  ├─ 「フォローアップ登録」──▶ DASH-06 フォローアップに追加
  ├─ 「ステージ変更」──▶ インラインドロップダウン（DASH-03上）
  └─ 「戻る」──▶ DASH-02 リード一覧 または DASH-01 パイプライン（遷移元に戻る）

DASH-04 自動化設定 (/settings)
  │
  ├─ タブ「スキャン」──▶ SET-01
  ├─ タブ「メール」──▶ SET-02
  └─ タブ「決済」──▶ SET-03
```

## 遷移図（営業パイプライン = ユーザーストーリーベース）

```
【自動営業フロー】

スキャンエンジン（自動実行）
  │
  ▼
LLMO未対策企業を検出
  │
  ▼
Firestore に新規リードとして保存
  │  ステージ: 「発見」
  ▼
フォーム自動探索
  │  ステージ: 「フォーム探索」→「フォーム発見」or「フォーム未発見」
  ▼
AIスコアリング（Gemini API）
  │  リードスコア算出 → 優先度ソート
  ▼
メールキュー投入
  │  ステージ: 「キュー」
  ▼
営業メール自動送信（SendGrid/Resend）
  │  ステージ: 「送信済」
  ▼
反応検知
  ├─ 開封あり ──▶ ステージ: 「開封」
  ├─ リンククリック ──▶ ステージ: 「興味あり」
  ├─ 返信あり ──▶ ステージ: 「商談」
  └─ 無反応（3日後）──▶ フォローアップキューへ
        │
        ▼
      フォローアップメール送信
        │
        ├─ 反応あり ──▶ 「商談」
        └─ 無反応 ──▶ 「休眠」（3ヶ月後に再スキャン対象）

商談
  │
  ├─ Stripe決済リンク送付
  ├─ 受注 ──▶ ステージ: 「受注」──▶ DASH-08 顧客・収益に追加
  └─ 失注 ──▶ ステージ: 「失注」──▶ 3ヶ月後に再アプローチ対象
```

## 新画面追加時のルール

1. 上記の画面一覧テーブルに ID・パス・画面名・認証要否・説明を追記する
2. 遷移図に遷移元（どこからこの画面に来るか）を必ず記載する
3. 遷移図に遷移先（この画面からどこに行けるか）を必ず記載する
4. サイドナビに表示する場合は「サイドナビ」セクションに追記する
5. 認証不要のページ（LP系）と認証必要のページ（ダッシュボード系）を混同しない

**遷移元が存在しない画面は作成してはいけない。**
